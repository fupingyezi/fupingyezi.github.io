算法修炼手册day1-哈希

2025-07-20 2025-07-29

前言：哈希

哈希即通过一系列函数映射（哈希函数）将待处理序列散列化，从而映射到不同的地址中，以地址key存储具体的值，访问值时只需知晓其对应的key值即可，是一种空间换时间的算法。
一般我们在解决哈希问题的时候最主要是确定键是什么，再考虑怎么映射

下面介绍两个JavaScript常用的处理哈希的方法

Set集合

Set 对象是值的合集（collection）。集合（set）中的元素只会出现一次，即集合中的元素是唯一的。

构造函数

常用内置实例方法

  • set.add(value)：在该集合中插入一个具有指定值的新元素，如果该 Set 对象中没有具有相同值的元素。
  • set.clear()：移除该集合中的所有元素。
  • set.delete(value)：从该集合中删除指定值，如果该值在集合中，返回true。
  • set.difference(other):接受一个集合并返回一个新的集合，其中包含当前集合中存在但给定集合中不存在的所有元素。
  • set.entries()：返回一个新的集合迭代器对象，该对象包含了此集合中每个元素的 [value, value] 数组，按插入顺序排列。Set 对象没有类似于 Map 对象中的 key。因此，为了保持 API 与 Map 对象类似，这里每个 _entry_ 的 _key_ 和 _value_ 都相同，所以返回的数组为 [value, value]。
  • set.forEach(callbackFn, [thisArg])：按插入顺序为该集合中的每个值执行一次提供的函数。
  • set.has(value)：返回一个布尔值来指示对应的值是否存在于该集合中。
  • set.intersection(other)：交集
  • set.isDisjointFrom(other)：返回一个布尔值来指示是否含有交集。
  • set.isSubsetOf(other)：是否是另一个集合的子集。
  • set.SupersetOf(other)：是否包含另一个集合。
  • set.keys()：等价于set.values()，返回一个新的set迭代器对象，该对象包含此集合中每个元素的值，按插入顺序排列。
  • set.symmetricDifference(other)：并集-交集。
  • set.union(other)：并集
  • set[Symbol.iterator]():迭代器对象。

常用内置实例属性

  • set.size：访问器属性返回此集中 （唯一） 元素的数量。

Map键值对

Map 对象是键值对的集合。Map 中的一个键只能出现一次；它在 Map 的集合中是独一无二的。Map 对象按键值对迭代——一个for...of循环在每次迭代后会返回一个形式为 [key, value] 的数组。

构造函数

常用内置实例方法

  • map.clear()：移除 Map 对象中所有的键值对。
  • map.delete(key)：移除 Map 对象中指定的键值对，如果键值对存在并成功被移除，返回 true，否则返回 false。调用 delete 后再调用 map.has(key) 将返回 false。
  • map.entries()：返回一个新的迭代器对象，其包含 Map 对象中所有键值对 [key, value] 二元数组，以插入顺序排列。
  • map.forEach(callbackFn,[thisArg])：以插入顺序为 Map 对象中的每个键值对调用一次 callbackFn。如果为 forEach 提供了 thisArg 参数，则它将作为每一次 callback 的 this 值。
  • map.get(key)：返回与指定的键 key 关联的值，若不存在关联的值，则返回 undefined。
  • map.has(key)：返回一个布尔值，用来表明 Map 对象中是否存在与指定的键 key 关联的值。
  • map.keys()：返回一个新的迭代器对象，其包含 Map 对象中所有元素的键，以插入顺序排列。
  • map.set(key, value)：在 Map 对象中设置与指定的键 key 关联的值，并返回 Map 对象。
  • map.values()：返回一个新的迭代对象，其中包含 Map 对象中所有的值，并以插入 Map 对象的顺序排列。
  • map.prototype[Symbol.iterator]()：返回一个新的迭代器对象，其包含 Map 对象中所有元素 [key, value] 二元数组，以插入顺序排列。

    常用内置实例属性

  • map.size：返回此 map 中元素的数量。

因为Object跟Map有点像的，所以我们有时也用Object来解决哈希问题。

一、两数之和

主考知识点：哈希，数组
1. 两数之和 - 力扣（LeetCode）


因为此题返回的不是两个整数，而是两个整数对应的下标，所以要以整数映射下标，因此选用map而不是set。
算法复杂度为O(n)

二、字母异位词分组

主考知识点：哈希表，字符串，排序，数组
49. 字母异位词分组 - 力扣（LeetCode）


解法一：字典序字符串标识一组字母异位词

因为每一组的字母异位词都可以重新排列已形成彼此，所以我们以字典序从小到大排序的字符串标识每一组字母异位词即可，我们简称为标识异位词，由于最后输出一个元素为数组的集合，所以要用每一个标识异位词映射每一组字母异位词数组，因此我们选择map而不是set。
算法复杂度为O(n+∑(k.length * n * log(n)))

解法二：官方解法字母计数法

由于互为字母异位词的两个字符串包含的字母相同，因此两个字符串中的相同字母出现的次数一定是相同的，故可以将每个字母出现的次数使用字符串表示，作为哈希表的键。
由于字符串只包含小写字母，因此对于每个字符串，可以使用长度为 26 的数组记录每个字母出现的次数。需要注意的是，在使用数组作为哈希表的键时，不同语言的支持程度不同，因此不同语言的实现方式也不同。
时间复杂度：O(n(k+∣Σ∣))，∣Σ∣=26

三、最长连续序列

主考知识点：哈希表，数组
128. 最长连续序列 - 力扣（LeetCode）


要达到o(n)的时间复杂度，肯定是不能排序的，所以我们考虑一种逻辑上的排序，是不是就想到了哈希表，对于两个连续的数字x、x+1，存储在哈希表内只需判断x+1在不在就行了。所以对于以每个数字为起点的连续序列，只需依次枚举x+1，x+2，…，x+y即可。
而如果要求最长连续序列，我们只需确定这个最长序列的起点在哪，然后枚举过去就能求出了，那怎么确定起点呢？
我们注意到，既然是起点，那肯定是没有前驱的，所以我们只需判断x-1在不在集合里就行啦。然后因为我们求长度是通过每一个起点枚举得出的，每一次求最大值就行了，就不需要map来标识了，所以我们用set。
时间复杂度：o(n)
