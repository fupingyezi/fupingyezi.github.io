MCP: Model Context Protocol 模型上下文协议

2026-02-04 2026-02-04

MCP(Model Context Protocol)模型上下文协议，2024.11由Anthropic开源，作为一个新标准，其用于连接AI助手到存储数据的地方，包括内容库、业务工具和开发环境。目的是为了帮助前沿模型生成更好、更相关的回复。

Model Context Protocol

MCP是一个开发标准，使开发者能够在其数据源和AI驱动工具之间建立安全的双向链接。直观的，我们可以将其理解为一个中间协议层，让开发者能够以一致的方式将各种数据源、工具和功能连接到AI模型，如下。

但其实这样说复杂了，我们换个更易懂的说法。
MCP其实核心分为三个组件：Host、Client和Server。各部分关系如下，均由MCP架构链接在一起：

我们简单地理解这一过程：

  • 首先你对你的智能体应用（Host）提出了一些询问：比如今天北京天气如何？又或者帮我整理一下我E盘下某某文件夹下的文档。
  • 然后其中LLM在两个问题下分别决定访问远端网络服务器来搜索天气以及访问系统文件系统。此时内置的MCP Client激活，来连接对应MCP Server。
  • 比如在上面两个问题中，前者会访问用于连接远程网络服务的Server，后者则会连接本地文件系统资源的Server。

对于这个过程，在提示词工程中有一个function_call的功能与之很相似，function_call就是给予LLM与外部工具连接的能力，比如外部网络API，知识库检索。当然这与tool_call也很相似。

而MCP服务就像是把这一层function_call/tool_call进一步封装起来，解耦部署在了外部服务器，而不会被单个平台所局限。

搭建一个自己的MCP服务器

so，了解上述前提之后，我们如何搭建一个MCP服务器呢？

1. 环境搭建

  • 初始化node.js项目
  • 下载MCP的sdk
  • 然后下载类型验证工具zod和一些node和ts环境的常用工具

然后我们主要介绍如何创建mcp server，关于整个sdk的教程可以看typescript-sdk/docs at main · modelcontextprotocol/typescript-sdk
。

2. server传输方式

sdk根据传输方式的不同给出了三种server：

  • Streamable HTTP：一般用于远程服务器，响应和请求通过HTTP POST/GET实现，可支持服务端到客户端的SSE。
  • HTTP+SSE：已经弃用的一种方式，仅用于向后兼容，请求使用HTTP，响应和通知通过单独的SSE连接。
  • stdio：用于本地、进程创建的集成，标准化输入输出。
